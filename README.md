Core Functionality:
1)Firebase Authentication (sign in / sign up)
2)Account overview & balance
3)Transaction history
4)Real‑time sync using Firebase
5)Local data caching with Room
6)Clean and modern UI using Jetpack Compose
7)MVVM architectural pattern

Technologies used:
1)Kotlin
2)Jetpack Compose
3)Firebase (Auth & Realtime)
4)Room Database
5)Android Studio

Project Notes:
1)UI: Uses Jetpack Compose for building modern, responsive UI layouts.
2)Offline persistence: Uses Room to cache local data for performance.
3)Firebase Realtime Sync: Keeps the user’s banking data updated in real time.

Architecture:
The app follows MVVM (Model‑View‑ViewModel) to separate UI, business logic, and data layers for better scalability and easier maintenance.
